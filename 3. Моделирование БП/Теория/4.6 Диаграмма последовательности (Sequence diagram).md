## 📘 Диаграмма последовательности (Sequence Diagram)

**Диаграмма последовательности** — это UML-диаграмма, которая показывает, **как объекты взаимодействуют друг с другом во времени**.

Она описывает порядок вызова методов и обмен сообщениями между участниками системы.

---

## 🔷 Основная идея

Если:

* Use Case → *что* делает система
* Class Diagram → *из чего* состоит система
* **Sequence Diagram → *как* выполняется процесс шаг за шагом**

---

## 🔷 Основные элементы

### 1️⃣ Участник (Participant)

Это объект или актор, участвующий в процессе.

Обозначается прямоугольником сверху и вертикальной линией жизни (lifeline):

```id="x7k2dn"
User        OrderService        PaymentService
 |               |                   |
 |               |                   |
```

---

### 2️⃣ Линия жизни (Lifeline)

Вертикальная пунктирная линия — время существования объекта.

---

### 3️⃣ Сообщение (Message)

Горизонтальная стрелка между участниками.

```id="m1q5sj"
User  →  OrderService : createOrder()
```

---

### 4️⃣ Возврат результата (Return)

Пунктирная стрелка назад.

```id="r8lv4b"
OrderService - - - > User : orderId
```

---

### 5️⃣ Активация (Activation)

Прямоугольник на линии жизни — период выполнения операции.

---

## 🔷 Пример: Оформление заказа

Участники:

* Пользователь
* OrderService
* PaymentService
* Database

Последовательность:

```id="p4n7ye"
1. User → OrderService : createOrder()
2. OrderService → PaymentService : processPayment()
3. PaymentService → Database : savePayment()
4. PaymentService → OrderService : paymentOK
5. OrderService → Database : saveOrder()
6. OrderService → User : orderConfirmed
```

Это показывает полный сценарий выполнения операции.

---

## 🔷 Дополнительные конструкции

### 🔁 Цикл (loop)

Используется для повторяющихся действий.

### ❓ Условие (alt)

Альтернативные ветви (например, успешная и неуспешная оплата).

### 🧩 opt

Необязательное действие.

---

## 🔷 Где применяется

✅ Проектирование бизнес-процессов
✅ Микросервисная архитектура
✅ Анализ логики системы
✅ Подготовка к разработке

---

## 🔷 Отличие от других UML-диаграмм

| Диаграмма | Показывает                     |
| --------- | ------------------------------ |
| Use Case  | Взаимодействие с пользователем |
| Class     | Структуру                      |
| Sequence  | Поведение во времени           |
| Activity  | Логику процесса                |
