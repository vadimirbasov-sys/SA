## 📘 Диаграмма компонентов (Component Diagram)

**Диаграмма компонентов** — это UML-диаграмма, которая показывает из каких **компонентов** состоит система и как они зависят друг от друга.

Она описывает **архитектуру на уровне модулей**, а не классов.

---

## 🔷 Что такое компонент?

Компонент — это самостоятельная часть системы:

* модуль
* библиотека
* сервис
* микросервис
* API
* подсистема

Обозначается прямоугольником со специальной иконкой:

```
┌──────────────────────┐
│   Payment Service    │  ⧉
└──────────────────────┘
```

---

## 🔷 Основные элементы

### 1️⃣ Компонент

Логический или физический модуль системы.

---

### 2️⃣ Интерфейс

#### Предоставляемый интерфейс (Provided Interface)

Обозначается кружком (lollipop):

```
( IOrderService )
        ○─── Order Service
```

#### Требуемый интерфейс (Required Interface)

Обозначается полуокружностью (socket):

```
Payment Service ───⊂ IOrderService
```

---

### 3️⃣ Зависимость (Dependency)

Пунктирная стрелка:

```
WebApp - - - > Auth Service
```

---

## 🔷 Пример: Интернет-магазин

Компоненты:

* `WebApp`
* `Order Service`
* `Payment Service`
* `Database`

Связи:

```
WebApp → Order Service
Order Service → Payment Service
Order Service → Database
```

Это показывает:

* веб-приложение использует сервис заказов
* сервис заказов использует оплату
* данные хранятся в базе

## 🔷 Отличие от других диаграмм

| Диаграмма  | Показывает            |
| ---------- | --------------------- |
| Class      | Структуру классов     |
| Object     | Конкретные объекты    |
| Component  | Архитектуру модулей   |
| Deployment | Физическое размещение |
