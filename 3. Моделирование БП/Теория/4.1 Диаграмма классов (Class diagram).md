## 📘 Диаграмма классов (Class Diagram)

**Диаграмма классов** — это один из основных видов UML-диаграмм, который описывает структуру системы через классы, их атрибуты, методы и связи между ними.

Используется при объектно-ориентированном проектировании для моделирования структуры программы.

---

## 🔷 Основные элементы

### 1️⃣ Класс

Изображается в виде прямоугольника из трёх частей:

```
---------------------
| Имя класса        |
---------------------
| атрибуты          |
---------------------
| методы            |
---------------------
```

**Пример:**

```
---------------------
| User              |
---------------------
| - id: int         |
| - name: string    |
---------------------
| + login()         |
| + logout()        |
---------------------
```

### Обозначения доступа:

* `+` — public
* `-` — private
* `#` — protected

---

## 🔷 Связи между классами

### 1️⃣ Ассоциация (Association)

Обычная связь между классами.
Пример: `User` — `Order` (пользователь делает заказ)

Обозначается простой линией:

```
User -------- Order
```

---

### 2️⃣ Наследование (Inheritance)

Один класс наследует другой.

Обозначается стрелкой с пустым треугольником:

```
        User
          ▲
          |
      Admin
```

`Admin` наследует `User`.

---

### 3️⃣ Агрегация (Aggregation)

Отношение "часть–целое" (слабая связь).

Обозначается пустым ромбом:

```
Car ◇------ Wheel
```

Колёса могут существовать отдельно от машины.

---

### 4️⃣ Композиция (Composition)

Сильная связь "часть–целое".

Обозначается закрашенным ромбом:

```
House ◆------ Room
```

Комната не существует без дома.

---

### 5️⃣ Зависимость (Dependency)

Временное использование одного класса другим.

Обозначается пунктирной стрелкой:

```
Order - - - > PaymentService
```

---

## 🔷 Пример простой диаграммы

Интернет-магазин:

* `User`
* `Order`
* `Product`
* `Payment`

Связи:

* User 1 — * Order
* Order 1 — * Product
* Order → Payment
