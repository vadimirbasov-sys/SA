### UML - язык, предназначенный для проектирования программных систем

Ключевая особенность **uml** - это объектно-ориентированный подход, где система представлена в виде набора взаимодействующих объектов, которые обладают собственными данными и поведением

### Основные элементы

**Класс** - Прямоугольник, разделенный на 3 секции (Название, Свойства, Методы)<br>
**Объект** - Конкретный экземпляр класса<br>
**Интерфейс** - Набор операций, которые класс должен реализовать<br>
**Действие** - Элементарный шаг в диаграмме активности<br>
**Состояние** - Ситуация в жизненном цикле объекта<br>
**Связи** - Различные линии, показывабщие, что объекты связаны (Ассоциация, Зависимость, Наследование, Агрегация)<br>

### Типы UML диаграмм

#### 1. Структурные диаграммы
* **Диаграмма классов** - Отображает классы, их атрибуты, методы, отношения
* **Диаграмма объектов** - Отображает объекты и значения их атрибутов в моменте
* **Диаграмма компонентов** -  Модули, библиотеки, исполняемые файлы.
* **Диаграмма развертывания** - Показывает на каких серверах/сетевых устройствах располагаются компоненты системы 

#### 2. Поведенческие диаграммы
* **Диаграмма деятельности (Activity diagram)** - блоксхема с описанием последовательности действий, ветвления. Похожа на BPMN.
* **Диаграмма последовательности (Sequence Diagram)** - показывает как объекты взаимодействуют в реальном времени. Нужна для описания API.
* **Диаграмма состояний (State Mashine Diagramm)** - Жизненный цикл одного объекта, показывает движение по статусам (Новый - Сформирован - Отгружен - Получен)
*   **Диаграмма прецедентов (Use Case Diagram)** - Показывает акторов и варианты использования системы
