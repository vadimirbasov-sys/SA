## 📘 Диаграмма прецедентов (Use Case Diagram)

**Диаграмма прецедентов** — это UML-диаграмма, которая показывает, **как внешние пользователи (акторы) взаимодействуют с системой**.

Она описывает **функциональные требования** к системе простым и понятным способом.

---

## 🔷 Основные элементы

### 1️⃣ Актор (Actor)

Это внешний участник системы:

* пользователь
* администратор
* внешняя система
* устройство

Обозначается человечком:

```
   👤
  User
```

---

### 2️⃣ Прецедент (Use Case)

Это действие или функция системы.

Обозначается овалом:

```
 ( Оформить заказ )
```

---

### 3️⃣ Граница системы

Система изображается прямоугольником, внутри которого находятся прецеденты.

```
+--------------------------------+
|        Интернет-магазин        |
|   (Оформить заказ)             |
|   (Оплатить заказ)             |
+--------------------------------+
```

---

## 🔷 Связи

### 1️⃣ Ассоциация

Линия между актором и прецедентом.

```
User -------- (Оформить заказ)
```

---

### 2️⃣ <<include>> (Включение)

Один прецедент **обязательно** включает другой.

```
(Оформить заказ) → <<include>> → (Проверить оплату)
```

---

### 3️⃣ <<extend>> (Расширение)

Дополнительное поведение при определённых условиях.

```
(Оплатить заказ) ← <<extend>> ← (Использовать купон)
```

---

### 4️⃣ Обобщение (наследование)

Акторы могут наследоваться:

```
      User
        ▲
        |
     Admin
```

---

## 🔷 Пример: Интернет-магазин

Акторы:

* Покупатель
* Администратор

Прецеденты:

* Регистрация
* Вход
* Просмотр товаров
* Оформление заказа
* Управление товарами

Связи:

* Покупатель → Регистрация, Вход, Оформление заказа
* Администратор → Управление товарами

## 🔷 Отличие от других диаграмм

| Диаграмма | Показывает              |
| --------- | ----------------------- |
| Use Case  | Что делает система      |
| Class     | Из чего состоит         |
| Component | Архитектуру             |
| Sequence  | Как выполняется процесс |
