## 📘 Диаграмма состояний (State Machine Diagram)

**Диаграмма состояний** — это UML-диаграмма, которая показывает, **в каких состояниях может находиться объект** и **как он переходит из одного состояния в другое** в ответ на события.

Она описывает поведение объекта во времени.

---

## 🔷 Когда используется

✅ Моделирование жизненного цикла объекта
✅ Описание бизнес-сущностей (заказ, заявка, договор)
✅ Проектирование автоматов
✅ Embedded-системы
✅ Backend-логика

---

## 🔷 Основные элементы

### 1️⃣ Начальное состояние (Initial State)

Закрашенный кружок:

```
●
```

---

### 2️⃣ Состояние (State)

Прямоугольник с закруглёнными углами:

```
[ Оплачен ]
```

Можно указывать действия внутри состояния:

```
[ Обработка ]
 entry / начать_проверку()
 exit  / сохранить_лог()
```

---

### 3️⃣ Переход (Transition)

Стрелка между состояниями.

Формат подписи:

```
событие [условие] / действие
```

Пример:

```
оплатаПодтверждена / отправитьЧек()
```

---

### 4️⃣ Конечное состояние (Final State)

Кружок с точкой:

```
◎
```

---

## 🔷 Пример: Жизненный цикл заказа

Состояния:

* Новый
* Оплачен
* Отправлен
* Доставлен
* Отменён

Последовательность:

```
●
 ↓
[ Новый ]
   ├─ оплата → [ Оплачен ]
   │               ↓
   │          отправка → [ Отправлен ]
   │                          ↓
   │                     доставка → [ Доставлен ] → ◎
   └─ отмена → [ Отменён ] → ◎
```

Это показывает полный жизненный цикл заказа.

---

## 🔷 Дополнительные возможности

### 🔹 Составные состояния

Состояние может содержать вложенные под-состояния.

### 🔹 Параллельные состояния

Объект может находиться одновременно в нескольких регионах состояний.

### 🔹 История (History state)

Позволяет вернуться к предыдущему под-состоянию.

---

## 🔷 Отличие от других диаграмм

| Диаграмма | Показывает                  |
| --------- | --------------------------- |
| Activity  | Поток действий              |
| Sequence  | Обмен сообщениями           |
| State     | Изменение состояний объекта |
| Use Case  | Функции системы             |
