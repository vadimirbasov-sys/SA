## 📘 Диаграмма деятельности (Activity Diagram)

**Диаграмма деятельности** — это UML-диаграмма, которая показывает **поток действий (workflow)** в системе или бизнес-процессе.

Она похожа на блок-схему, но использует нотацию UML.

---

## 🔷 Для чего используется

✅ Описание бизнес-процессов
✅ Моделирование алгоритмов
✅ Анализ логики работы системы
✅ Документирование требований

---

## 🔷 Основные элементы

### 1️⃣ Начальное состояние (Start)

Обозначается закрашенным кружком:

```
●
```

---

### 2️⃣ Действие (Activity / Action)

Прямоугольник с закруглёнными углами:

```
[ Проверить данные ]
```

---

### 3️⃣ Переход (Control Flow)

Стрелка от одного действия к другому.

```
[Ввод данных] → [Проверка]
```

---

### 4️⃣ Решение (Decision)

Ромб — условное ветвление:

```
        [Оплата прошла?]
           /        \
        да          нет
```

---

### 5️⃣ Параллельность (Fork / Join)

Толстая линия — разделение или объединение потоков:

```
      ────────
      |      |
   [Действие1] [Действие2]
```

---

### 6️⃣ Конечное состояние (End)

Обозначается кружком с точкой:

```
◎
```

---

## 🔷 Пример: Оформление заказа

Последовательность действий:

```
● 
 ↓
[Выбрать товар]
 ↓
[Добавить в корзину]
 ↓
[Оформить заказ]
 ↓
[Оплатить]
 ↓
[Оплата прошла?]
   ├─ Да → [Подтвердить заказ] → ◎
   └─ Нет → [Показать ошибку] → ◎
```

---

## 🔷 Отличие от других диаграмм

| Диаграмма | Показывает          |
| --------- | ------------------- |
| Use Case  | Функции системы     |
| Sequence  | Обмен сообщениями   |
| Activity  | Логику процесса     |
| State     | Изменение состояний |

---

## 🔷 Где применяется

* Бизнес-процессы (BPM)
* Банковские системы
* Интернет-магазины
* Производственные процессы
* Алгоритмы программ

---
