## 📘 Диаграмма пакетов (Package Diagram)

**Диаграмма пакетов** — это UML-диаграмма, которая показывает **структурную организацию системы на уровне пакетов** и зависимости между ними.

Она используется для логической группировки элементов (классов, интерфейсов, компонентов) и отображения связей между этими группами.

---

## 🔷 Что такое пакет?

**Пакет (Package)** — это контейнер, который объединяет связанные элементы модели.

Примеры:

* Пакеты в Java (`com.shop.service`)
* Пространства имён в C#
* Модули проекта
* Подсистемы

---

## 🔷 Обозначение пакета

Пакет изображается как папка:

```id="a7h3jk"
  _________
 |         |
 |  Service|
 |_________|
```

Можно показывать вложенность:

```id="zq8l2p"
Shop
 ├── Model
 ├── Service
 └── Controller
```

---

## 🔷 Основные связи

### 1️⃣ Зависимость (Dependency)

Пунктирная стрелка:

```id="l1s8rt"
Controller - - - > Service
Service - - - > Model
```

Это означает:

* `Controller` использует `Service`
* `Service` использует `Model`

---

### 2️⃣ Импорт (Import)

Пакет может импортировать элементы другого пакета.

---

### 3️⃣ Вложенность

Пакеты могут содержать другие пакеты.

---

## 🔷 Пример: Архитектура интернет-магазина

Пакеты:

* `Presentation`
* `BusinessLogic`
* `DataAccess`
* `Common`

Связи:

```id="s5w9nb"
Presentation → BusinessLogic
BusinessLogic → DataAccess
Все пакеты → Common
```

Это отражает слоистую архитектуру.

---

## 🔷 Отличие от других UML-диаграмм

| Диаграмма  | Показывает             |
| ---------- | ---------------------- |
| Class      | Классы и их связи      |
| Component  | Модули системы         |
| Package    | Группировку элементов  |
| Deployment | Размещение на серверах |
