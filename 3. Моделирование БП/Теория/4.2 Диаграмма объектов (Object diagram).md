## 📘 Диаграмма объектов (Object Diagram)

**Диаграмма объектов** — это UML-диаграмма, которая показывает *конкретные экземпляры классов* и их связи в определённый момент времени.

Если **диаграмма классов** показывает структуру системы в целом, то диаграмма объектов — это «снимок» работающей системы.

---

## 🔷 Что отображает диаграмма объектов?

* Конкретные объекты (экземпляры классов)
* Их текущие значения атрибутов
* Связи между объектами

---

## 🔷 Обозначение объекта

Объект изображается прямоугольником:

```
objectName : ClassName
-----------------------
атрибут = значение
```

Имя объекта подчёркивается (в UML).

---

## 🔷 Пример

Предположим, есть классы:

* `User`
* `Order`
* `Product`

### 📌 Диаграмма объектов (конкретный момент времени)

```
user1 : User
----------------
id = 1
name = "Иван"

order15 : Order
----------------
number = 15
date = 27.02.2026

productA : Product
----------------
name = "Ноутбук"
price = 80000
```

Связи:

```
user1 -------- order15
order15 ------ productA
```

Это означает:

* Иван сделал заказ №15
* Заказ содержит ноутбук

---

## 🔷 Отличие от диаграммы классов

| Диаграмма классов    | Диаграмма объектов         |
| -------------------- | -------------------------- |
| Показывает типы      | Показывает экземпляры      |
| Описывает структуру  | Показывает состояние       |
| Не содержит значений | Содержит реальные значения |

---

## 🔷 Когда используется

✅ Анализ конкретного сценария
✅ Отладка логики системы
✅ Демонстрация работы модели
✅ Проверка корректности связей
